## Key Features

### ✅ Authentication
- User registration with email verification

- JWT token management
- Session persistence
- Password reset via OTP

### ✅ Public Routes

- `/admin/*` - Admin dashboard

### ✅ Role-Based Access
- **Customer** - Browse, buy, view tickets

## Common Questions

**Q: Where's the login page?**
A: `/login` - No more mock users to pick from!

**Q: How do I create a test account?**
A: Go to `/register` → Fill form → Verify email with OTP → Login

**Q: Can I browse events without logging in?**
A: Yes! Homepage is public. Only protected features need login.

**Q: Where's the OTP code?**
A: Check backend logs when you register. (Email service can be configured later)

**Q: How long does session last?**
A: JWT token valid for 1 day. Token stored in localStorage.

**Q: How do I test different roles?**
A: Register accounts, then set their role in database for testing.
